[2026-06-03 02:40:44] Bug Fix: Infinite loop in PDF Download step (Full AI-driven Scoping Mode)
- Issue: The application entered an infinite loop of downloading PDFs and reloading the page. This occurred because the auto-download check evaluated ALL articles (including excluded ones) for missing download status. Since excluded articles are never downloaded, their status remains empty, causing the check to persistently pass and re-trigger the download process upon every reload.
- Resolution: Modified the logic in app.py to specifically check the included_df (Included articles only) for missing download statuses rather than the entire df.

[2026-06-03 02:44:27] Bug Fix: Empty Title and Abstract from PubMed Search
- Issue: In Tab 1, PMIDs were being saved to the database immediately after the ID fetch but before the full XML was fetched and parsed. Since the database insertion used INSERT OR IGNORE, the subsequent insertion of the fully parsed data (containing titles and abstracts) was ignored because the PMIDs already existed. This caused many articles to have blank titles and abstracts in the database.
- Resolution 1: Removed the premature PMID save block in app.py so the data is only saved once after full parsing.
- Resolution 2: Fixed deduplication logic in pubmed_parser.py to prevent it from accidentally dropping multiple distinct articles that had empty titles.
- Note: The reason empty title/abstract articles were marked as Included is because the LLM prompt explicitly instructs it to default to Included if the abstract is missing or vague, which is intended behavior for conservative screening.
